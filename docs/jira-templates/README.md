# PACE Jira Issue Templates

Jira does not use file-based templates — these are configured in your Jira project via
**Project Settings → Issue Types → Description Templates** (Jira Cloud Next-gen) or via
the **Description** field default on each issue type (Jira Classic).

Copy the templates below into your Jira project configuration.

---

## Template 1: PACE HOLD Escalation (Issue Type: Bug / Priority: High)

> Recommended label: `pace-hold`
> PACE will auto-create these via `JiraTrackerAdapter.open_escalation_issue()`.

```
h2. Escalated HOLD — Day [DAY]

PACE could not resolve this HOLD after 2 retries. Human intervention required.

*Blocker:* [HOLD_REASON]

----

h3. Story Card
{code:yaml}
[story.md content]
{code}

h3. FORGE Handoff (last attempt)
{code:yaml}
[handoff.md content]
{code}

h3. GATE Report (last attempt)
{code:yaml}
[gate.md content]
{code}

----

h3. To Resume
# Resolve the blocker described above
# Close this ticket
# Set the PACE_PAUSED CI/CD variable to *false*
# The PACE loop will re-run Day [DAY] on the next scheduled trigger

_Note: PACE_PAUSED was automatically set to true when this ticket was opened._

_Generated by PACE Orchestrator_
```

## Template 2: PACE Advisory Finding (Issue Type: Task / Priority: Medium)

> Recommended label: `pace-advisory`
> PACE will auto-create these via `JiraTrackerAdapter.push_advisory_items()`.

```
h2. PACE Advisory — [AGENT] Day [DAY]

[AGENT] raised [N] advisory finding(s) on Day [DAY] that were not resolved after one retry.
These are non-blocking but must be cleared on the next clearance day.

h3. Findings
* [finding 1]
* [finding 2]

----

h3. Resolution
# Address each finding before the next clearance day
# On a clearance day, SENTINEL/CONDUIT will verify and clear resolved items
# Unresolved items on a clearance day become blockers (HOLD)

_Generated by PACE Orchestrator — Day [DAY]_
```

## Template 3: PACE Story Ticket (Issue Type: Story)

> Recommended label: `pace-story`
> PACE will auto-create these via `JiraTrackerAdapter.push_story()`.

```
h2. PACE Story — Day [DAY]

*Target:* [story target]

h3. Acceptance Criteria
* [ ] [criterion 1]
* [ ] [criterion 2]

----

h3. Full Story Card
{code:yaml}
[plan.yaml story card content]
{code}

----

_Opened automatically by PACE Orchestrator — Day [DAY]_
```

## Environment Variables

Configure these in your CI/CD platform to control Jira integration:

| Variable | Default | Description |
| --- | --- | --- |
| `JIRA_URL` | — | Base URL, e.g. `https://mycompany.atlassian.net` |
| `JIRA_EMAIL` | — | Atlassian account email for Basic auth |
| `JIRA_TOKEN` | — | API token from id.atlassian.com → Security → API tokens |
| `JIRA_PROJECT_KEY` | — | Project key, e.g. `ENG` or `PACE` |
| `JIRA_HOLD_ISSUE_TYPE` | `Bug` | Issue type for HOLD escalations |
| `JIRA_ADVISORY_ISSUE_TYPE` | `Task` | Issue type for advisory findings |
| `JIRA_STORY_ISSUE_TYPE` | `Story` | Issue type for story tickets |
| `JIRA_HOLD_PRIORITY` | `High` | Priority for HOLD issues |
| `JIRA_ADVISORY_PRIORITY` | `Medium` | Priority for advisory issues |
